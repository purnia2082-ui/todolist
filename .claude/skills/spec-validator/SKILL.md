# Spec Validator Skill

## 역할
PRD와 TECH_SPEC을 기준으로 구현된 코드가 스펙에 정확히 일치하는지 3단계로 검증하는 도구입니다.

## 전제조건
- **docs/PRD.md 필수**
- **docs/TECH_SPEC.md 필수**
- **src/ 폴더 필수**: 검증할 코드가 존재해야 함
- 하나라도 없으면 → 해당 단계를 먼저 완료하도록 안내

## 3단계 검증 프로세스

### Stage 1: PRD 일치 검증
PRD의 수용 기준을 하나씩 코드와 대조

```markdown
## Stage 1: PRD 일치 검증

### 기능 1: [기능명]
| 수용 기준 | 판정 | 근거 |
|----------|------|------|
| [기준 1] | ✅ PASS / ❌ FAIL | [코드 위치 또는 미구현 사유] |
| [기준 2] | ✅ PASS / ❌ FAIL | [근거] |
| [기준 3] | ✅ PASS / ❌ FAIL | [근거] |

### 기능 2: [기능명]
[동일 구조]

### 기능 3: [기능명]
[동일 구조]
```

### Stage 2: TECH_SPEC 일치 검증
TECH_SPEC의 구현 명세와 실제 코드를 대조

```markdown
## Stage 2: TECH_SPEC 일치 검증

### 파일 구조
| TECH_SPEC 명세 | 실제 파일 | 판정 |
|---------------|----------|------|
| src/types/[type].ts | [존재 여부] | ✅ / ❌ |
| src/hooks/[hook].ts | [존재 여부] | ✅ / ❌ |
| src/components/[comp].tsx | [존재 여부] | ✅ / ❌ |

### 함수/인터페이스
| TECH_SPEC 명세 | 실제 구현 | 판정 |
|---------------|----------|------|
| [함수명/타입명] | [구현 여부] | ✅ / ❌ |

### API 엔드포인트 (해당 시)
| TECH_SPEC 명세 | 실제 구현 | 판정 |
|---------------|----------|------|
| [Method] [Endpoint] | [구현 여부] | ✅ / ❌ |
```

### Stage 3: 코드 품질 검증
코드 품질 기준 체크

```markdown
## Stage 3: 코드 품질 검증

| 항목 | 판정 | 비고 |
|------|------|------|
| TypeScript 타입 안전성 | ✅ / ❌ | [any 사용 여부 등] |
| 에러 처리 | ✅ / ❌ | [try-catch, 에러 바운더리 등] |
| 접근성 (a11y) | ✅ / ❌ | [aria 속성, 시맨틱 HTML 등] |
| 하드코딩 여부 | ✅ / ❌ | [매직 넘버, 문자열 등] |
| 컴포넌트 단일 책임 | ✅ / ❌ | [과도한 책임 여부] |
```

## 종합 리포트 형식

```markdown
# 스펙 검증 리포트

> 검증 일시: [날짜]
> 프로젝트: [프로젝트명]

## 종합 결과

| 단계 | 결과 | 점수 |
|------|------|------|
| Stage 1: PRD 일치 | ✅ / ⚠️ / ❌ | [N/M] |
| Stage 2: TECH_SPEC 일치 | ✅ / ⚠️ / ❌ | [N/M] |
| Stage 3: 코드 품질 | ✅ / ⚠️ / ❌ | [N/M] |
| **종합** | **[판정]** | **[총점]%** |

## 불일치 항목 상세

### [불일치 항목 1]
- **스펙**: [PRD/TECH_SPEC에 명시된 내용]
- **실제**: [실제 구현 상태]
- **차이**: [무엇이 다른지]
- **개선 제안**: [수정 방향]

### [불일치 항목 2]
[동일 구조]

## 개선 권고사항

### 우선순위 높음 (PRD 불일치)
1. [개선 항목]

### 우선순위 중간 (TECH_SPEC 불일치)
1. [개선 항목]

### 우선순위 낮음 (품질 개선)
1. [개선 항목]
```

## 판정 기준

| 종합 점수 | 판정 | 의미 |
|----------|------|------|
| 100% | ✅ PERFECT | 스펙 완벽 일치 |
| 80-99% | ✅ PASS | 경미한 차이, 수용 가능 |
| 60-79% | ⚠️ WARNING | 주요 불일치 존재, 수정 필요 |
| 0-59% | ❌ FAIL | 심각한 불일치, 재구현 권고 |

## 주의사항
- 주관적 의견이 아닌 스펙 기반으로만 판정
- 모든 FAIL 항목에 구체적 근거 제시
- 개선 제안은 실행 가능한 수준으로 구체적으로
- 스펙에 없는 추가 구현은 감점 대상이 아님 (단, 스펙 누락은 감점)
